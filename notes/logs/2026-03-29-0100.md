## Root Cause Analysis

### Problem: All 65536 NRS2 pixels flagged DO_NOT_USE

Previous sessions had identified that the photom step was the issue (NaN fill for
out-of-range wavelengths). This session revealed the EARLIER cause: **flat_field**.

**Step-by-step diagnosis:**
1. With default flat: 0/65536 good pixels
2. With `flat_field: skip: True`: 32076/65536 good pixels
3. The flat field references have `FAST_VARIATION` (wavelength-dependent correction) that
   only covers 0.97–1.89 µm (NRS1 range). Out-of-range → DO_NOT_USE for NRS2.

**Flat references (default CRDS, context jwst_1464.pmap):**
- `fflat_0154.fits`: FAST_VARIATION covers only 0.962–1.840 µm
- `sflat_0154.fits`: FAST_VARIATION covers only 0.970–1.890 µm
- `dflat_0002.fits`: covers 0.6–5.3 µm (fine)

**Fix: Parlanti extended flat references:**
- `fflat_0105.fits` (G140M): 0.970–5.500 µm, `slit_name=ANY`
- `fflat_0091.fits` (G235M): 1.660–5.500 µm, `slit_name=ANY`
- `sflat_0191.fits` (G140M/NRS2): 0.970–4.994 µm, `slit_name=ANY`
- `sflat_0192.fits` (G235M/NRS2): 1.660–5.992 µm, `slit_name=ANY`
All are NRS_IFU references (EXP_TYPE=NRS_IFU) but `slit_name=ANY` allows use for FS.

### Secondary problem: extract_1d from s2d gives NPIXELS=0

After flat_field was fixed, the s2d had valid WHT and SCI data in the aperture rows
(rows 7–11 of 18, WHT=44.2, SCI peak=36.8 MJy). But extract_1d still gave NPIXELS=0.

**Fix: skip `resample_spec`, extract directly from cal.**
When extract_1d runs on the cal (MultiSlit model), it correctly produces 1953/2048
valid flux points. Root cause of s2d issue not fully identified but likely related to
the extended-wavelength photom correction not propagating correctly through drizzle.
